## Mission: Code Review

Your task is to review code changes for quality, security, and style.

### Workflow:

1.  **Analyze Diff**:

    - Read the provided diff or file content.
    - Identify the language and context.

2.  **Check Criteria**:

    - **Correctness**: Does the code do what it claims?
    - **Security**: Are there any vulnerabilities (SQL injection, XSS, etc.)?
    - **Style**: Does it follow standard conventions for the language?
    - **Performance**: Are there any obvious bottlenecks?

3.  **Provide Feedback**:
    - List specific issues with line numbers.
    - Suggest improvements or refactorings.
    - Be constructive and clear.
